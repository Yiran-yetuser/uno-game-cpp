
## 技术栈

- **语言**：C++17
- **构建系统**：CMake ≥ 3.16
- **图形界面**：Qt 6（Widgets 模块）
- **测试**：纯 C++ 控制台程序（无 Qt 依赖）

## 功能实现

- [ ] 经典 UNO 玩法（108 张牌，数字、行动、万能牌）
- [ ] 人机对战（ 人类 vs  AI）
- [ ] AI 自动决策（基于优先级策略）
- [ ] 卡牌叠加规则（+2 可叠加 +2，+4可以接在+2后，+2不可以叠加在+4后）
- [ ] 惩罚跳过机制（无法应对 +N 时摸牌并跳过回合）
- [ ] 破产出局（手牌 ≥25 张立即淘汰）
- [ ] 排名制胜负（全部出局后按顺序确定名次）
- [ ] Qt 图形界面（卡牌渲染、拖拽/点击出牌、动画效果）
- [ ] 独立测试程序（验证核心逻辑）

## 总体设计

项目遵循 **前后端分离** 原则：

- **后端** (`src/core`)：负责所有游戏规则、状态管理、AI 决策。完全不依赖 Qt，可脱离界面在控制台中运行和测试。
- **前端** (`src/ui`)：基于 Qt6 Widgets，负责画面渲染和用户交互。通过 `Game` 类的公开方法发送操作，通过回调函数接收状态更新。
- **接口**：`Game` 作为门面类，提供 `humanPlayCard()`, `humanDrawCard()`, `humanChooseColor()` 等操作，并通过 `std::function` 回调通知 UI。

### 核心模块

| 模块   | 职责                                       |
| ------ | ------------------------------------------ |
| Card   | 卡牌 ID 映射、颜色/数字/动作、可出牌判定   |
| Dealer | 牌堆初始化、洗牌、发牌、抽牌、回收弃牌     |
| Judger | 规则判定：合法性、叠加、破产、胜负         |
| Player | 手牌容器、玩家名称                         |
| AiPlayer | AI 选牌策略（调用 Judger）              |
| Round  | 回合状态机：方向、惩罚累计、淘汰顺序       |
| Game   | 组合上述模块，对外提供统一接口             |

## 游戏规则

### 基本规则
- 108 张牌，包含 4 种颜色（红、绿、蓝、黄）的数字 0-9、行动牌（跳过、反转、+2）以及万能牌（变色、+4）。
- 双方各发 7 张手牌，翻开弃牌堆第一张。
- 轮流出牌，须与弃牌堆顶颜色一致、数字/符号相同，或打出万能牌。
- 无牌可出时必须摸牌，若摸到的牌可立即打出则自动打出。
- 率先出完所有手牌的玩家获胜，游戏按出完顺序排名。

### 进阶规则
- **叠加惩罚**：
  - `+2` 牌只能被 `+2` 或 `+4` 牌叠加。
  - `+4` 牌可被任何功能牌叠加（`+2` 或 `+4`）。
  - `+2` 不可在 `+4` 后叠加。
- **惩罚跳过**：
  - 当一方需要承受 `+N` 惩罚且无法叠加时，摸取对应数量的牌，并跳过本回合出牌阶段（视为同时吃了一记“跳过”）。
- **破产淘汰**：
  - 任意玩家手牌数达到或超过 25 张时，立即淘汰出局，排名垫底。
