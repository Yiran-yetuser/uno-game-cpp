📖 项目简介

本项目实现了一个完整的 UNO 纸牌游戏，提供两种独立的玩法界面：底层游戏规则用 C++ 实现为可复用的类库， 同一个核心规则被两种不同的 UI 前端调用。

🏗️ 核心类与算法

整个项目采用经典的面向对象分层架构，从下到上依次是：

Card（数据）→ Deck（牌堆）→ Player / AIPlayer（玩家）→ Game（调度）
↑
console_ui.cpp / uno-game.html（UI 层）

1. Card 类 — 数据基石

class Card
include/card.h · src/card.cpp · 78 行

用 enum class 严格枚举 Color 和 CardType，杜绝魔法字符串。

enum class Color  { Red, Yellow, Green, Blue, Wild };
enum class CardType { Number, Skip, Reverse, DrawTwo, Wild, WildDrawFour };

每张牌只占 8 字节（1 字节 Color + 1 字节 Type + 4 字节 int + 2 字节 padding），整副 108 张牌 ≈ 864 字节，内存极省。

核心算法：canPlayOn()

bool Card::canPlayOn(const Card &top) const {
    if (type == CardType::WildDrawFour) return true;
    if (type == CardType::Wild && top.type != WildDrawFour) return true;
    if (color == top.color) return true;
    if (type == Number && top.type == Number && number == top.number) return true;
    if (type == top.type && type != Number) return true;
    return false;
}

四步短路判断：① 万能+4 必可出 ② 万能牌（非 +4 之上）可出 ③ 同色 ④ 数字牌同号 ⑤ 同类型非数字牌。

2. Deck 类 — 牌堆管理

class Deck
include/deck.h · src/deck.cpp · 83 行

内部维护 drawPile（抽牌堆）和 discardPile（弃牌堆）两个 vector<Card>。

算法 1：buildFullDeck() — 标准 108 张生成

// 4 色 × 数字牌: 0×1 + 1-9×2 = 19 张/色  → 76 张
// 4 色 × 功能牌: Skip/Reverse/DrawTwo ×2 = 6 张/色  → 24 张
// Wild × 4 + WildDrawFour × 4 = 8 张
// 合计 76 + 24 + 8 = 108 张

算法 2：shuffle() — Fisher-Yates 洗牌

void Deck::shuffle(std::vector<Card> &pile) {
    std::mt19937 rng(static_cast<unsigned>(time(nullptr)));
    std::shuffle(pile.begin(), pile.end(), rng);
}

使用 std::shuffle，底层是 O(n) 的 Fisher-Yates，每张牌出现在每个位置的概率严格相等 ≈ 1/108。

算法 3：reshuffleDiscard() — 弃牌回收

void Deck::reshuffleDiscard() {
    if (discardPile.size() <= 1) return;
    Card top = discardPile.back();      // 保留顶牌
    drawPile.insert(drawPile.end(),
                    discardPile.begin(),
                    discardPile.end() - 1);
    discardPile.clear();
    discardPile.push_back(top);
    shuffle(drawPile);
}

关键点：保留弃牌堆顶牌（它是当前生效的底牌），其余所有牌回收并重新洗牌。这样抽牌堆耗尽时游戏可以无限继续。

3. Player / AIPlayer — 玩家基类与 AI

class Player → class AIPlayer : public Player
include/player.h, ai_player.h · src/player.cpp, ai_player.cpp

Player 只负责手牌容器：drawCard()、playCard(idx)、getHandCount()，逻辑最纯粹。

AIPlayer 通过继承复用 Player 的全部能力，只新增 chooseCard() 一个决策方法：

int AIPlayer::chooseCard(const Card &topCard) const {
    for (int i = 0; i < hand.size(); ++i) {
        if (hand[i].canPlayOn(topCard)) return i;
    }
    return -1;
}

AI 类本身是最朴素的"出第一张合法牌"策略，而 更智能的策略写在 Game::gameLoop() 里的 AI 回合分支中（见下文）。

4. Game 类 — 顶层调度与规则仲裁

class Game
include/game.h · src/game.cpp · 367 行

状态字段

Deck deck;                 // 牌堆
std::vector<Player*> players;  // [人类, AI1, AI2, AI3]
Card topCard;              // 弃牌堆顶
Color currentColor;        // 实际生效的颜色（万能牌选色后可能 ≠ topCard.color）
int currentPlayerIdx;     // 当前玩家下标
int direction;            // 1=顺时针, -1=逆时针
int drawStack;            // 叠加罚牌总数

核心算法 1：canPlayCard() — 出牌合法性仲裁

bool Game::canPlayCard(Player *p, const Card &play) const {
    // 罚牌叠加态：+4 之上只能叠 +4，+2 之上可叠 +2 或 +4
    if (drawStack > 0) {
        if (drawStack >= 4) return play.type == CardType::WildDrawFour;
        else return play.type == DrawTwo || play.type == WildDrawFour;
    }
    // 正常态：委托给 Card::canPlayOn
    return play.canPlayOn(topCard);
}

这一层比 Card::canPlayOn 更严格：当场上存在叠加罚牌时，强制只能出对应类型，避免非法叠加（+4 上不能叠 +2）。

核心算法 2：executeEffect() — 出牌后效果

void Game::executeEffect(const Card &played) {
    switch (played.type) {
    case CardType::Skip:    nextPlayer();                break;
    case CardType::Reverse: direction *= -1;            break;
    case CardType::DrawTwo:  drawStack += 2;             break;
    case CardType::WildDrawFour: drawStack += 4;         break;
    }
}

Skip 的"跳过"实现得很巧妙：先调用一次 nextPlayer() 跳到下一位，然后本回合正常走 nextPlayer()，等于多走了一格，等价于"被跳过"。

核心算法 3：gameLoop() 主循环

翻开局顶牌（优先数字）→ 随机先手 → 进入循环
↓
玩家分支：cin 读序号 → canPlayCard 校验 → playCard → 选色（如万能）→ executeEffect
↓
AI 分支：智能策略（见下） → 选色（选手牌最多色）→ executeEffect
↓
罚牌叠加处理 → nextPlayer() → 循环直到 checkWin()

核心算法 4：AI 智能出牌策略（Game::gameLoop 内嵌）

// AI 智能评分，按"分数最高"选择出牌
int score = 0;
if (card.type == CardType::Number)       score = card.number;
if (card.type == CardType::DrawTwo)     score = 20;
if (card.type == CardType::Skip)       score = 20;
if (card.type == CardType::Reverse)    score = 20;
if (card.type == CardType::Wild)       score = 50;
if (card.type == CardType::WildDrawFour) score = 50;
// 同分时优先出当前 currentColor 的数字
if (card.color == currentColor) score += 2;

策略：优先甩大点 / 功能牌 / 万能牌，其次才出同色数字牌，保留小点数作为后手。这比 AIPlayer::chooseCard 的"出第一张合法"更聪明。

核心算法 5：AI 万能牌选色

// AI 出完万能/万能+4 后，选手牌里数量最多的那个颜色
int cntR=0, cntY=0, cntG=0, cntB=0;
for (auto &c : ai->getHand()) {
    switch (c.color) {
        case Red: cntR++; break;
        case Yellow: cntY++; break;
        ...
    }
}
currentColor = (cntR>cntY&&cntR>cntG&&cntR>cntB) ? Red : ...;

5. UI 层 — 两套独立前端

console_ui.cpp (C++ 控制台)

  • 设置 SetConsoleOutputCP(936) + GBK 编码，与 src/ 的 GBK 源码一致
  • 使用 Windows Console API：SetConsoleCursorPosition 实现任意坐标输出
  • SetConsoleTextAttribute 控制颜色（12=红 / 14=黄 / 10=绿 / 11=蓝）
  • _getch() 替代 cin 实现"按键即响应"，避免回车确认
  • 四方位布局：AI 一号在左、AI 二号在上、AI 三号在右、玩家在下
  • ASCII 字符画框（+、-、|、=）保证任意代码页都不乱码
  • toString() 返回中文（如"红5"、"蓝跳过"），控制台正确显示

uno-game.html (HTML 浏览器)

  • 使用 Cards/ 目录中 60+ 张真实 UNO 卡图，按 color_name 命名映射
  • uno-game-standalone.html 把所有图片转 base64 内嵌，发布版零依赖
  • CSS 用 position: absolute 配合 top/left/right/bottom 实现四方位
  • JavaScript GameState 对象保存 players / topCard / currentColor / direction 等
  • Web Audio API 生成出牌/抽牌/反转/UNO 等音效

🎮 UNO 玩法规则

卡牌示例规则
数字牌 (0-9) 0~9 同色或同号可接
Skip 跳过 ⊘ 下一位本轮跳过
Reverse 反转 ⇄ 出牌顺序反转（2 人时等同跳过）
Draw Two +2 +2 下一家罚抽 2 张，可继续叠加 +2 / +4
Wild 万能 W 选下一轮颜色
Wild +4 +4 选色 + 罚抽 4 张，仅在 +4 之上可继续叠 +4
注意：+4 叠加规则 —— 已有 +4 时不能再出 +2，只能出 +4 继续叠加或选择受罚。这是 UNO 的标准规则。

💡 关键设计决策

分层架构：Card/Deck/Player/AIPlayer/Game 五层，每层只依赖下层。修改任一层不会波及其他层。
currentColor 与 topCard.color 解耦：万能牌打出后，用户选的"下家颜色"与 topCard.color 不同，单独用 currentColor 字段记录，UI 渲染与出牌规则都基于 currentColor。
GBK 编码贯穿：src/ 源文件全部 GBK 编码，控制台设 CP936，与 Windows 中文环境天然兼容，无需 BOM、无需 pragma。
惩罚叠加用整数计数：用 drawStack 累加 +2/+4，无需复杂的"链式"数据结构，规则逻辑写在 canPlayCard 的第一段。
